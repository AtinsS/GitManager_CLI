# 🚀 Git Manager

> Простой инструмент для управления несколькими Git-репозиториями без лишних команд

**Платформа:** Windows  
**Язык:** Batch Script

---

## 💭 Зачем я его сделал?
У меня несколько репозиториев, и я устал каждый раз писать одни и те же Git-команды.
```
git add .
git commit -m "fix"
git push origin main
```

Хотелось чего-то простого: открыл, выбрал действие и всё работает.
Так и появился **🚀 Git Manager**.

---
### 🧑‍💻 Это для тебя, если:

- 📦 у тебя несколько репозиториев  
- 😵 устал писать одни и те же Git-команды  
- ⚡ хочешь делать commit + push в одно нажатие  
- 🔘 нужно быстро обновлять все проекты  

---
## 🧰 Что умеет проект?

- 📁 Управление несколькими репозиториями
- 📦 Распределение проектов по группам
- ⬆️ Commit + Push одним действием
- ⬇️ Массовое обновление всех репозиториев одной кнопкой
- 📊 Статус репозиториев прямо в меню
- 🌿 Работа с ветками
- 🔄 Clone / Pull / Push без CLI
- 📜 История изменений (git log)
- 🤖 Авто-коммиты по таймеру
- 🌐 Поддержка языков с автоопределением при запуске (RU / EN)
---

## ⚙️ Быстрый старт

1. Установи [Git](https://git-scm.com/)
    
2. Скачай проект, либо склонируй репозиторий
    
3. Запусти: `GIT-MANAGER.BAT`

4. Кайфуй 🚀

---

### Действия внутри репозитория

```batch
  ════════════════════════════════════════════════════════════
  Репозиторий: Repo1  Ветка: dev
  Статус: ✅ чисто
  ════════════════════════════════════════════════════════════

  1. Git status (проверить состояние)
  2. Git pull (обновить)
  3. Git add + commit + push (с комментарием)
  4. Перейти в меню отмены изменений
  5. Просмотр истории (git log)
  6. Git merge (слияние веток)
  7. Git merge --abort (отменить слияние)
  8. Показать ветки для слияния
  9. Создать ветку
  10. Переключить ветку
  11. Авто-коммиты (каждые N минут)
  0. Вернуться в главное меню

 →
```

---

### ❓ Почему Batch Script?
- не требует установки
- работает из коробки
- лёгкий и быстрый
